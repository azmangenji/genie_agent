================================================================================
DETAILED ERROR ANALYSIS - CMD-012
================================================================================

SYMPTOM:
  Error: extra positional option '_sel1015576' (CMD-012)
  Error: Failure in flow change during 'before' evaluation - (DSO-6804)

ROOT CAUSES IDENTIFIED:

  1. set_max_area 0 $cells
     - This command is DESIGN-LEVEL, not cell-level
     - When passed a large cell collection, FC expands all cell names
     - Command line becomes too long, causing CMD-012
     - SOLUTION: Remove or use at design level only

  2. set_boundary_optimization $cells all
     - This command only works on HIERARCHICAL cells, not leaf cells
     - When collection contains leaf cells, causes MSG-2117 warning
     - Then CMD-012 when collection expansion fails
     - SOLUTION: Filter to hierarchical cells only

AFFECTED PERMUTON VALUES (umccmd_dcq_arbiter_pipeline):
  - "restructure" -> FAILS (triggers set_max_area)
  - "both"        -> FAILS (triggers set_max_area)
  - "none"        -> OK (returns early, skips problematic code)
  - "balance"     -> OK (do_restructure=false, skips set_max_area)

AFFECTED FILES (set_boundary_optimization):
  - procs/umccmd_dcq_arbiter_procs.tcl
  - procs/umccmd_timing_counter_procs.tcl
  - procs/umccmd_arb_safe_reg_procs.tcl
  - procs/umccmd_pgt_opt_procs.tcl
  - procs/umcdat_aes_pipeline_procs.tcl
  - procs/umcdat_key_retime_procs.tcl
  - procs/umcdat_xts_pipeline_procs.tcl
  - procs/umcdat_ecc_read_procs.tcl
  - procs/umcdat_deep_pipeline_procs.tcl
  - procs/umcdat_beq_tx_procs.tcl
  - procs/umcdat_rddatpipe_procs.tcl

AFFECTED FILES (set_max_area):
  - procs/umccmd_dcq_arbiter_procs.tcl
  - procs/umcdat_ecc_read_procs.tcl


================================================================================
FIX PATTERNS
================================================================================

FIX 1: set_boundary_optimization - filter to hierarchical cells only

  BEFORE:
    set_boundary_optimization $cells all

  AFTER:
    # Filter hierarchical cells only for set_boundary_optimization
    # (leaf cells cannot have boundary optimization set - causes CMD-012)
    set hier_cells [filter_collection $cells "is_hierarchical == true"]
    if {[sizeof_collection $hier_cells] > 0} {
        set_boundary_optimization $hier_cells all
        puts "  Applied boundary optimization to [sizeof_collection $hier_cells] hierarchical cells"
    }

FIX 2: set_max_area - remove or use at design level

  BEFORE:
    set_max_area 0 $cells

  AFTER:
    # Note: set_max_area removed - causes CMD-012 with large cell collections
    # Area constraints should be set at design level if needed
    # Use: set_max_area 0  (without cell argument) at design level
